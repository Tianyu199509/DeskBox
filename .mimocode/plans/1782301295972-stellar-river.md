# 设置页面重构方案

## 设计原则

1. 遵循 Windows Fluent Design 原生风格（NavigationView、设置卡片、ToggleSwitch、DropDownButton）
2. 主页面只放"理解成本低"的设置（开关、下拉选择）
3. 滑块、数字输入等细调项收进钻入式子页面（与随记设置相同的面包屑导航模式）
4. 9 个导航项合并为 6 个

## 导航结构

### 1. 常规
- 语言（下拉）
- 开机自启（开关）
- 双击打开文件（开关，从"交互"移入）
- 显示引导页（按钮）

### 2. 外观

**主页面 — 摘要卡片**：
- 主题（下拉）
- 系统强调色（开关）+ 自定义强调色（色块选择）
- 格子圆角（下拉）
- **"显示详细设置"齿轮按钮** → 钻入子页面

**子页面 — 面包屑：外观 > 显示详细设置**：
- 格子透明度（滑块 + 输入框）
- 默认宽度（输入框）
- 默认高度（输入框）
- 图标大小（滑块 + 输入框）
- 文字大小（滑块 + 输入框）
- 水平间距（滑块 + 输入框）
- 垂直间距（滑块 + 输入框）
- 文件名宽度（滑块 + 输入框）
- 快捷方式箭头（开关）
- 显示文件扩展名（开关）
- 隐藏快捷方式扩展名（开关，依赖"显示扩展名"）
- 列表详情（开关）
- 图片剪贴板（开关）

### 3. 功能格子（结构不变）
- 主页面：随记开关 + 齿轮按钮
- 子页面：随记全部详细设置

### 4. 高级

**动画**（卡片组）：
- 动画效果（下拉）
- 动画速度（下拉）

**快捷键**（卡片组）：
- 全局快捷键（开关）
- 快捷键绑定（按钮，从"交互"移入）

**存储**（卡片组）：
- 拖放行为（下拉）
- 托管路径（按钮 + 路径显示）
- 存储管理（已有内容，从"存储"移入）

### 5. 维护（不变）
- 重置设置
- 下载依赖

### 6. 关于（不变）
- 版本、理念、仓库

## XAML 结构变更

### 外观主页面
参考随记的摘要卡片模式（SettingsWindow.xaml:1057-1095）：
- 一个 `SettingsGroupStyle` 卡片包含主题、强调色、圆角
- 卡片底部或右侧放齿轮按钮
- 点击齿轮 → 切换到 `AppearanceDetailSection` 子页面

### 外观子页面
参考随记子页面模式（SettingsWindow.xaml:1098-1128）：
- 面包屑导航：`外观 > 显示详细设置`
- 返回按钮触发 `AppearanceBreadcrumbBackButton_Click`
- 所有滑块设置放在一个 `SettingsGroupStyle` 卡片内

### 高级页面
三个卡片组用分隔线区分，不需要子页面（总共 7 项，扁平展示即可）

## 不改动的部分

- 功能格子的主页面和子页面结构
- 维护、关于页面
- 所有已有的 Style 定义
- 所有 ViewModel 绑定属性
- Windows Fluent Design 原生控件使用方式

## 验证

1. 外观主页面：只看到主题、强调色、圆角 + 齿轮按钮，无滑块
2. 外观子页面：点击齿轮后进入，面包屑可返回，所有滑块设置可见
3. 高级页面：动画、快捷键、存储三个卡片组平铺
4. 交互页面不再存在（内容已分散到常规和高级）
5. 存储页面不再独立存在（合并到高级）
6. 动画页面不再独立存在（合并到高级）
