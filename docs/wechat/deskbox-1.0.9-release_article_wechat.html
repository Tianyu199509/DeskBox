先简单说下 DeskBox 是做什么的

如果你是第一次看到 DeskBox，可以把它理解为一个 Windows 桌面整理工具。

它主要做三件事：

  1. 收纳格子

把文件拖进桌面格子里，让桌面不再堆满临时文件。

  1. 映射文件夹

把常用文件夹直接映射成桌面格子，不移动原文件，也方便快速打开。

  1. 随记格子

保存常用文本、链接、截图和最近复制内容，适合临时记录和快速复用。

DeskBox 的目标不是替换 Windows 桌面，而是在原生桌面上补一层更轻的整理能力。

1.0.9 主要更新

这次更新不是单点功能，而是把很多基础体验一起整理了一遍。

设置页面重新整理

设置页做了一次结构重排，减少了顶层分类，把常用设置重新归到更清楚的位置。

同时调整了很多控件：

  • 下拉选择改成原生 ComboBox

  • 数字输入改成原生 NumberBox

  • 开关补充“开 / 关”文字

  • 外观、随记等入口改成更接近 Windows 设置的钻入式体验

  • 新增托盘图标样式：跟随系统、彩色、黑色、白色

这部分主要是为了让设置页更像一个工具该有的样子，而不是一堆功能临时堆在一起。

格子内新增排序

格子里现在支持按照类似 Windows 资源管理器的方式排序：

  • 名称

  • 大小

  • 项目类型

  • 修改日期

同一个排序项重复点击，可以切换升序和降序。每个格子的排序状态也会单独保存。

这个功能不大，但对日常找文件会舒服很多。

【图片占位：格子右键菜单里的排序方式截图】

右键菜单更清楚了

之前普通格子、映射格子、随记的右键菜单里，有一些操作混在一起。

这次做了拆分：

  • 标题栏右键更偏格子管理：新建、锁定、重命名、删除、打开文件夹、更改映射路径

  • 内容区右键更偏文件操作：添加文件、粘贴、新建文件夹、视图切换、排序、刷新

  • 随记标题栏右键保留随记相关操作

简单说，就是尽量让你点哪里，就出现和哪里相关的操作。

随记继续打磨

随记这一版也改了不少细节：

  • 记录 / 固定 / 最近三个 Tab 切换更直接

  • 去掉了复杂的 Tab 动画，减少卡顿和状态冲突

  • 标题栏按钮和悬浮按钮尺寸更接近普通格子

  • 单击内容复制的反馈更稳定

  • 随记关闭、显示、层级行为更接近普通格子

  • 随记总开关关闭后，剪贴板监听也会一起停止

随记以后还会继续优化，但这一版至少把基础交互先拉回稳定状态。

拖拽和层级稳定性

这次还集中修了一批底层体验问题：

  • 增强部分 Windows 版本下资源管理器拖拽到格子的兼容性

  • 空格子也能正确接收拖放

  • 收纳格子和映射文件夹的拖拽提示区分开

  • 优化 F7 / 托盘唤起后的临时置顶逻辑

  • 修复随记和普通格子偶尔层级不同步的问题

  • 菜单关闭后，不再让单个窗口自己强制置底，而是交给统一逻辑判断

这里也补一个排查提示：如果你遇到“从资源管理器拖文件到 DeskBox 时显示禁止符号，或者怎么都拖不进去”，可以先检查一下 DeskBox 是否被设置成了“以管理员身份运行”。

DeskBox 日常使用不需要管理员权限。Windows 资源管理器通常是普通权限运行，如果 DeskBox 被拉高到管理员权限，Windows 可能会直接阻止资源管理器把文件拖进 DeskBox。遇到这种情况，可以右键 DeskBox 快捷方式或 DeskBox.exe，进入属性里的“兼容性”，确认没有勾选“以管理员身份运行这个程序”，然后完全退出 DeskBox 再重新打开。应该可以解决一部分这类问题。另外win10的用户有很多，目前确实还没解决，主要是因为自己机器是win11的，加上工作有点忙，只能晚上下班了抽空修修bug。短期内还是以win11为主。win10有很多效果特性也没办法加，这个是架构问题。

文件重命名和图标显示修复

这次也修复了两个比较烦人的问题：

  • 格子内文件 / 文件夹重命名时，中文输入法不能稳定切换

  • 按 Esc 取消重命名时，不应该保存当前内容

  • 启动后部分文件图标偶尔显示异常，刷新后才正常

现在文件重命名改成了更稳定的编辑方式，图标加载也增加了重试逻辑。

新用户引导精简

新用户引导的文案和设置项也重新精简了一版，更偏 Windows 风格，减少一上来就让用户理解太多概念。

还有很多东西想做

DeskBox 现在还是一个早期工具。

它的方向我会继续保持克制：不接管桌面，不替代资源管理器，不做一套新的复杂桌面系统。

后面我更想把它做成一个稳定的桌面轻组件层：

  • 文件格子继续增强整理体验

  • 随记继续优化记录、搜索、图片和复用

  • 后续可能加入 Todo、监控、快捷动作等更多功能格子

  • 底层会逐步抽出统一的交互层级、外壳和内容组件

我希望它最后能像一个很自然的 Windows 桌面扩展，而不是一个很重的“桌面替代品”。
