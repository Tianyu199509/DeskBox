# DeskBox 文件格子与桌面整理指南

文件格子是 DeskBox 最核心的功能。它不是虚拟文件库，而是桌面上的真实文件夹入口。

本指南会说明文件实际存在哪里、哪些操作会改变真实文件，以及如何用少量格子建立稳定的桌面分类。

> [图片占位：一个收纳格子和一个映射格子并排。标题清楚显示「待处理」和「当前项目」，格子内使用真实文件图标]

## 收纳格子

收纳格子适合接收新文件。

创建后，DeskBox 会在收纳根目录下建立对应的真实文件夹。把文件拖入格子时，根据设置执行复制或移动。

典型用途

- 待处理文件
- 截图
- 下载临时内容
- 安装包与压缩包
- 当前项目素材
- 等待归档的办公文档

收纳格子不是只存在于 DeskBox 中。右键格子标题，可以在资源管理器中打开对应文件夹。

## 映射文件夹

映射文件夹适合快速访问已经存在的目录。

它不会创建副本，也不会改变目录位置。格子中显示的文件与资源管理器中是同一批真实文件。你在格子内执行重命名、移动或删除时，真实目录会同步变化。

典型用途

- 当前项目目录
- 代码仓库
- 设计素材库
- 常用模板
- 下载目录
- OneDrive 中的工作目录

映射的价值是缩短访问路径，不是把目录变成只读预览。

## 如何选择

| 需求 | 使用方式 |
| --- | --- |
| 文件刚来到电脑，需要一个归宿 | 收纳格子 |
| 已有目录结构，不希望移动 | 映射文件夹 |
| 希望拖入后来源文件消失 | 收纳格子，并将拖入行为设为移动 |
| 只想在桌面快速查看项目目录 | 映射文件夹 |
| 希望卸载后仍能直接访问 | 两者都可以，底层都是普通文件夹 |

## 创建和命名

从托盘右键菜单创建收纳格子，或者使用「映射文件夹」选择已有目录。

格子名称应该表达用途，而不是文件类型。比起「PDF」「图片」「文档」，更推荐

- 待处理
- 本周项目
- 客户交付
- 参考素材
- 常用模板

用途型命名更接近日常动作。自动叠放可以继续在一个用途格子内部按文件类型整理。

双击标题可以重命名。重命名收纳格子时，应留意对应真实文件夹的处理提示；映射格子的显示名称与真实路径概念不同，以界面实际提示为准。

## 拖入文件

默认拖入行为是复制。

- 复制，来源保留，格子目录产生一份副本。
- 移动，来源消失，文件进入格子目录。

可以在 `设置 > 文件格子 > 收纳与路径` 中调整全局默认值。映射文件夹的拖放遵循目标真实目录的 Windows 文件操作语义。

拖入时请观察鼠标和格子提示，确认当前是复制、移动还是不允许操作。跨磁盘、权限差异和目标目录权限都可能影响 Windows 最终行为。

## 拖出文件

从格子把文件拖到桌面、资源管理器、聊天窗口或其他支持 Windows 拖放的软件。

拖出不是「导出」。文件本来就是普通文件。目标软件决定接收方式，例如聊天软件可能上传文件，资源管理器可能复制或移动。

## 打开、选择和批量操作

- 双击文件，使用 Windows 默认应用打开。
- Ctrl 加点击，添加或取消单个选择。
- Shift 加点击，选择连续范围。
- 在空白处拖动，使用选择框选择多个项目。
- 右键选中项目，执行复制、剪切、重命名、删除或打开文件位置。

删除、移动和重命名会作用于真实文件。批量操作前应确认选择范围。

## 图标视图和列表视图

图标视图适合图片、设计素材和需要视觉识别的内容。

列表视图适合办公文档、安装包和需要查看名称、类型、大小、日期的内容。是否显示详细信息可以在文件显示设置中调整。

可以按名称、大小、类型或修改时间等方式排序。具体可用项取决于当前版本界面。

如果图片预览让格子显得太乱，可以开启「图片文件仅显示图标」。这不会修改图片，只改变展示方式。

## 自动叠放

当一个格子开始出现十几个甚至几十个项目时，可以开启自动叠放。

叠放按类型、日期或自定义扩展名规则整理视觉分组，不会移动真实文件。详细配置参考 [文件自动叠放与 QuickLook](06-file-stacks-and-quicklook.md)。

## 文件格子的胶囊状态

收起后，文件胶囊会保留格子名称和项目数量。拖文件靠近胶囊时，格子可以临时展开，让你确认落点。

胶囊宽度支持保持同宽或独立调整，调整时也会显示左右对齐参考线。详细说明参考 [胶囊模式与组合排列](05-capsule-mode.md)。

## 推荐的桌面分类

不要一开始创建十几个格子。推荐先使用四个

1. 待处理，所有临时内容先进入这里。
2. 当前项目，映射正在进行的项目目录。
3. 常用资料，映射模板、证件扫描或高频文档目录。
4. 输出与交付，放已经生成、等待发送或归档的成果。

使用一到两周后，再根据真实频率拆分。分类太细会让「应该放哪里」变成新的负担。

> [图片占位：四格推荐布局。用箭头展示文件从「待处理」进入「当前项目」，完成后进入「输出与交付」]

## 数据注意事项

- 映射格子内的删除会删除真实文件，不是只从格子隐藏。
- 收纳格子的移动模式会移除来源位置文件。
- DeskBox 数据备份主要保护配置、记录和附件关系，不等同于备份所有映射目录。
- 重要文件应继续使用 OneDrive、文件历史记录、版本控制或其他正式备份。
- 修改收纳根目录前，先完成备份并阅读迁移提示。
