# DeskBox 常见问题与故障排查

遇到问题时，先确认当前版本和系统环境。本文对应 DeskBox 1.3.0、Windows 11 x64。

建议排查前先备份重要数据，并记录问题发生步骤、屏幕数量、缩放比例和当前设置。

> [图片占位：诊断与维护页面，标出备份与恢复、数据健康和兼容性诊断入口]

## 安装时提示文件正在使用

1.3.0 安装器会主动关闭正在运行的 DeskBox。如果仍然出现重试、忽略、取消

1. 从托盘退出 DeskBox。
2. 打开任务管理器，确认 `DeskBox.exe` 和更新助手已经结束。
3. 等待几秒后选择重试。
4. 确认使用的是 1.3.0 最新安装包，而不是旧安装器。

不要选择忽略后继续覆盖正在使用的核心文件，可能导致版本混合。

## 安装器需要下载运行时

DeskBox 安装包不内置 .NET 10 Desktop Runtime 和 Windows App Runtime 2.2。目标电脑缺少时需要联网下载。

下载失败时检查网络、代理、防火墙和系统时间。也可以从微软官方渠道先安装对应 x64 运行时，再重新运行 DeskBox 安装包。

已经存在兼容运行时时，安装器会跳过，不会重复把它们安装进应用目录。

## 文件拖不进格子

最常见原因是权限级别不一致。

1. 右键 DeskBox 快捷方式或程序属性。
2. 打开兼容性页面。
3. 取消「以管理员身份运行此程序」。
4. 完全退出 DeskBox，再重新启动。

继续检查

- 目标格子是否锁定或只允许当前操作类型。
- 映射目录是否有写入权限。
- 文件是否被其他软件独占。
- 来源是否为微信、浏览器等虚拟文件拖放，来源应用是否真的提供文件内容。
- 拖动的是文本、文件还是快捷方式，当前格子是否支持该类型。

进入 `设置 > 诊断与维护 > 兼容性诊断` 可以检查常见拖放环境问题。

## 快捷方式图标空白或错误

快捷方式图标可能来自目标程序、图标资源或 Windows Shell 缓存。Steam 游戏、Notepad++ 等应用的快捷方式还可能依赖安装位置或协议目标。

排查

1. 在资源管理器中确认 `.lnk` 快捷方式本身图标正常。
2. 确认目标程序或 Steam 游戏仍然安装在原位置。
3. 删除格子中的失效快捷方式，重新从开始菜单或安装目录创建。
4. 刷新格子或重启 DeskBox，让 Shell 图标重新加载。

如果资源管理器中也空白，应先修复 Windows 图标缓存或快捷方式目标。

## F7 按下没有反应

检查

- DeskBox 是否正在托盘运行。
- `设置 > 快捷与交互 > 全局快捷键` 是否开启。
- F7 是否被游戏、IDE、显卡覆盖层或笔记本功能键占用。
- 重新录制其他组合后是否生效。
- DeskBox 是否以不同权限级别运行。

快捷键冲突时，Windows 可能拒绝注册，但程序本身仍然运行。可以先用托盘左键验证显示和隐藏功能。

## 按 F7 隐藏后无法再次唤起

先使用托盘左键尝试唤起。如果托盘也无反应，完全退出并启动最新版本。

如果格子实际已经显示但在屏幕外，参考下一节。频繁在旧安装目录和开发版本之间切换时，应确认任务管理器中只有一个 DeskBox 进程，快捷方式指向当前安装位置。

## 格子跑到屏幕外

常见触发场景

- 拔掉外接显示器
- 更改主显示器
- 调整分辨率或缩放
- 远程桌面连接与断开
- 展开动画或拖动中发生显示器变化

DeskBox 会根据显示器拓扑和锚点尝试把格子恢复到可见工作区。先重启最新版并等待显示器状态稳定，再使用 F7 或托盘唤起。

仍然不可见时，不要立即删除应用数据。先导出或复制当前数据，再提交屏幕布局、缩放比例和日志用于定位。

## 胶囊悬停不展开

检查

1. 胶囊总开关是否开启。
2. 当前格子是否单独设置为点击展开或保持展开。
3. 鼠标是否停在标题中间区域，而不是左侧图标或右侧按钮区。
4. 悬停响应是否设为防误触，等待时间较长。
5. 当前是否正在拖动、调整大小、编辑标题或打开菜单。

如果点击展开和收回一次后悬停才恢复，确认正在运行 1.3.0 最新版本，并在单格设置中恢复跟随全局后重试。

## 格子展开后不自动收起

自动收起会避开可能导致内容丢失或误操作的状态，例如

- 鼠标仍在格子或有效标题区域内
- 正在拖入文件或文本
- 正在调整大小
- 详情页处于编辑状态
- 菜单、弹窗或交互层尚未结束

先把鼠标完整移出格子并等待收起延迟。仍然无法收起时，点击收起按钮，并检查该格子是否有单独展开方式覆盖。

## 展开或收起时位置变化

确认宽度关系

- 保持同宽，胶囊和展开状态宽度联动。
- 独立调整，两种状态分别保存。

展开窗口应以胶囊边角为锚点。调整展开状态右边框不应带动左边，调整左边框不应改变右边固定边界。

如果位置持续漂移，记录胶囊所在屏幕边缘、拖动方向、展开尺寸和多屏缩放。暂时切换为保持同宽和点击展开，可以减少变量。

## 展开动画闪烁、出现黑底或半透明占位

先确认使用最新版本。1.3.0 的显示与隐藏动画移动真实窗口，不应在终点提前绘制整层占位。

继续检查

- Windows 是否关闭透明效果。
- 是否处于远程桌面或显卡驱动异常状态。
- 当前材质是否在系统环境中回退。
- 多个不同缩放显示器之间是否正在移动。
- 关闭动画后问题是否消失。

关闭动画可以作为排查手段。如果只有特定材质和动画组合出现问题，记录具体选项和视频。

## 材质看起来没有区别

云母、云母 Alt 和亚克力依赖 Windows DWM 合成。以下环境可能让差异变小或回退

- Windows 透明效果关闭
- 节能模式或图形能力受限
- 远程桌面
- 旧系统版本
- 背景过于单一

选择标准亚克力并使用有明暗变化的壁纸更容易观察差异。材质浓度只调整材质层次，不等于透明度。

纯色始终保持 100% 不透明，透明度滑块会隐藏。

## 设置下拉框没有显示当前值

确认关于页面显示 1.3.0。关闭并重新打开设置，切换一次语言后观察选项是否重新本地化。

不要使用旧版本曾经存在的页面恢复按钮。1.3.0 已移除容易造成选项集合状态异常的页面恢复入口，全局重置位于诊断与维护。

如果所有页面同时出现空值，保留设置文件和截图，不要立即全局重置，以便定位本地化或选项集合问题。

## QuickLook 按 Space 没有反应

确认

1. QuickLook 已安装。
2. QuickLook 已经运行。
3. 当前选中的是存在的文件或目录。
4. 焦点位于文件格子，而不是重命名输入框或其他控件。

DeskBox 不会自动启动 QuickLook，也没有集成开关。QuickLook 未运行时没有反应属于预期行为。

## 音乐格子没有内容

音乐格子依赖 Windows 系统媒体会话。

检查播放器是否正在播放，以及播放器是否支持系统媒体控制。可以先从 Windows 快捷设置或媒体面板确认系统能否看到歌曲信息。

浏览器网页播放器可能受网站、浏览器媒体设置和播放状态影响。DeskBox 本身不是播放器，不会扫描本地音乐库。

## 天气无法定位或刷新

检查网络连接、自动定位设置和城市选择。自动定位不准确时，改用手动城市。

刷新间隔不是实时天气推送。刚切换单位、城市或网络后，可以手动刷新并等待请求完成。

如果只有某个城市失败，记录城市名称、系统语言和时间；如果所有城市失败，检查网络、代理和服务可用性。

## 待办提醒没有出现

检查任务日期、提醒时间、DeskBox 运行状态和 Windows 通知权限。Windows 专注模式可能隐藏通知。

重复任务的下一次提醒依赖当前实例是否正确完成和生成后续任务。先在详情中确认下一次日期。

## 待办或随记附件失效

关联原路径模式下，文件移动、改名、删除或外部磁盘断开都会导致失效。

进入附件健康检查，确认属于

- 原路径缺失
- 托管附件缺失
- 孤立托管附件

找到原文件后重新添加。重要临时文件以后可以使用复制到 DeskBox 模式。

## 备份无法恢复

不要修改 ZIP 内部文件后继续恢复。完整性清单会检测变化。

检查

- 文件是否完整下载或复制。
- 是否为 DeskBox 支持的备份版本。
- ZIP 是否能正常打开。
- 磁盘空间是否足够。
- 设置页是否把快照标记为不可读取。

恢复失败时，保留原 ZIP、当前数据目录和恢复前快照。不要连续尝试多个来源不明的备份覆盖当前状态。

## 托盘菜单出现滚动条或高度不稳定

1.3.0 的托盘弹窗使用适合当前内容的稳定最小高度，正常情况下应直接展示全部操作。

如果第一次打开明显较矮、第二次才恢复，记录 Windows 缩放、任务栏位置、屏幕工作区和截图。不要通过拖动或更改系统文本大小掩盖问题，这通常与首次测量时机有关。

## 如何提交有效问题

提交问题时尽量包含

- DeskBox 版本
- Windows 版本
- 显示器数量、分辨率和缩放
- 复现步骤
- 期望结果与实际结果
- 截图或短视频
- 相关设置值
- 是否能稳定复现
