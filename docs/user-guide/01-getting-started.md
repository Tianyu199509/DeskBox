# DeskBox 安装与首次配置

本指南帮助新用户从下载安装到创建第一个格子。完成后，你应该能够用 F7 显示或隐藏全部格子，并理解收纳目录、收纳格子和映射文件夹的区别。

> [图片占位：DeskBox 首次启动后的完整桌面，至少展示一个文件格子、一个待办和一个随记]

## 系统要求

推荐环境

- Windows 11 64 位
- .NET 10 Desktop Runtime x64
- Windows App Runtime 2.2 x64
- 普通 Windows 用户权限

DeskBox 围绕 WinUI 3 和 Windows 11 的窗口、材质、圆角、拖放、通知与媒体会话能力设计。Windows 10 不属于当前推荐环境，部分视觉和交互可能无法完整工作。

DeskBox 不需要以管理员身份运行。不要在快捷方式兼容性设置中勾选「以管理员身份运行此程序」。资源管理器通常以普通权限运行，DeskBox 如果被提升为管理员权限，Windows 可能阻止两者之间的文件拖放。

## 下载安装包

Direct 安装包可以从官网或 GitHub Release 获取。

- 官网：<https://deskbox.fun>
- GitHub Release：<https://github.com/Tianyu199509/DeskBox/releases/tag/v1.3.0>
- 文件名：`DeskBox_Setup_1.3.0_x64.exe`

Microsoft Store 渠道由商店负责安装和更新。两个渠道的核心功能一致，但更新方式和部分渠道展示内容可能不同。

从 GitHub 下载时，可以同时下载 `.sha256` 校验文件。DeskBox 1.3.0 x64 正式安装包的 SHA-256 为

```text
9AD3AC78829DD5A5500B6F6C57537F17C87451A440AD861C82F11EA3ADE10AF8
```

## 安装过程

安装器不会把 .NET 10 和 Windows App Runtime 2.2 的完整安装程序打进 DeskBox 安装包。它会先检查目标电脑

- 依赖已经存在，直接跳过。
- 缺少依赖，联网下载对应运行时。
- DeskBox 正在运行，先关闭进程，再替换文件。

这种方式可以控制安装包体积，也避免每次升级重复安装相同运行时。

## 首次启动引导

首次启动会依次帮助你完成以下配置。

### 选择收纳根目录

收纳根目录保存所有收纳格子背后的真实文件夹。默认路径适合大多数用户；如果系统盘空间紧张，也可以选择其他本地磁盘。

建议把收纳根目录固定到资源管理器的快速访问。这样即使 DeskBox 没有显示，也能直接从资源管理器访问格子中的文件。

收纳根目录不是数据库。里面仍然是普通文件夹和文件。

### 选择外观

第一次使用建议保留以下默认思路

- 主题跟随 Windows。
- 材质使用云母。
- 显示密度使用标准或舒适的默认状态。
- 边框和圆角先跟随默认值。

确认日常阅读和拖放都舒服以后，再进入外观设置调整透明度、材质浓度、边框颜色和密度。

### 启用功能格子

待办、随记、音乐和天气都可以单独开启。关闭某项后，创建入口会隐藏，但不会影响其他类型格子。

新用户推荐先开启待办和随记。音乐依赖当前播放器是否支持 Windows 系统媒体会话，天气需要网络。

### 设置快捷键和开机启动

默认全局快捷键是 F7。

- 按一次，显示或唤起全部格子。
- 再按一次，隐藏全部格子。

如果 F7 与其他软件冲突，可以进入 `设置 > 快捷与交互 > 全局快捷键` 重新录制。

开机启动适合希望 DeskBox 长期常驻托盘的用户。如果只是先体验，可以暂时关闭，确认工作流后再开启。

## 创建第一个格子

右键系统托盘中的 DeskBox 图标，选择创建格子。

建议第一个格子命名为「待处理」。它用于承接暂时不知道该归档到哪里，但不希望继续散落在桌面上的文件。

把一张测试图片拖进去。默认拖入行为是复制，因此原文件会保留。希望拖入后清理来源位置时，可以进入文件格子设置，把收纳拖入行为改为移动。

接着使用「映射文件夹」选择一个常用目录。映射不会复制目录，它直接展示真实位置中的内容。

> [动图占位：托盘创建「待处理」格子，拖入图片，再映射一个项目文件夹]

## 新用户推荐初始配置

| 项目 | 推荐值 | 原因 |
| --- | --- | --- |
| 全局快捷键 | F7 | 单键显示或隐藏全部格子 |
| 收纳拖入 | 复制 | 初次使用更不容易误移动文件 |
| 附件保存 | 关联原路径 | 不重复占用磁盘空间 |
| 胶囊模式 | 先关闭 | 先熟悉完整格子的操作 |
| 待办预览 | 2 行 | 扫读与信息密度较平衡 |
| 随记预览 | 3 行 | 能看到足够上下文 |
| Enter 行为 | Enter 换行，Ctrl+Enter 保存 | 符合多行编辑习惯 |
| 自动叠放 | 先关闭 | 文件较多后再开启更容易理解效果 |

## 覆盖升级

从旧版本升级时，可以直接运行新安装包覆盖安装。正常情况下会保留

- 格子配置、位置和尺寸
- 收纳目录中的文件
- 待办、随记及其附件关系
- 外观、快捷键和其他设置
- 胶囊与展开状态的保存数据

重要版本升级前，建议进入 `设置 > 诊断与维护 > 备份与恢复` 导出 ZIP 备份。覆盖安装不会替代数据备份。

## 卸载前检查

卸载前建议完成三件事

1. 在设置中确认当前收纳根目录。
2. 用资源管理器打开目录，确认重要文件可独立访问。
3. 导出一份 DeskBox 数据备份。

收纳格子中的普通文件不会因为应用消失就变成不可读格式，但卸载程序可能询问是否删除本地应用数据。操作前应看清选项。
