# DeskBox 备份、恢复与数据维护指南

DeskBox 长期保存格子配置、待办、随记、附件关系和大量个性化设置。开始长期使用后，应尽早理解哪些内容属于 DeskBox 数据，哪些仍然需要独立文件备份。

> [图片占位：设置中的备份与恢复页面。展示导出、恢复、立即创建快照、快照列表和状态信息]

## 备份保护什么

DeskBox 数据备份主要覆盖

- 应用设置
- 格子配置、位置和尺寸
- 待办数据
- 随记数据
- DeskBox 管理的附件数据与引用关系
- 恢复所需的清单和完整性摘要

备份不会自动复制所有外部映射目录，也不等于整台电脑备份。

如果文件格子映射了 `D:\Projects`，DeskBox 备份会保护映射关系和格子配置，但项目目录本身仍应由版本控制、云盘或其他备份方案保护。

## 手动导出 ZIP

进入 `设置 > 诊断与维护 > 备份与恢复`，选择导出并指定保存位置。

DeskBox 不会直接一边读取正在变化的数据目录一边压缩。它会

1. 创建稳定的临时快照。
2. 检查源文件在复制过程中是否变化。
3. 验证快照中的关键数据。
4. 从快照生成 ZIP。
5. 写入包含文件 SHA-256 的清单。
6. 清理临时目录。

如果数据在复制过程中持续变化，DeskBox 会重试或停止，而不是静默生成前后状态混杂的压缩包。

## 恢复备份

恢复前先关闭正在编辑的任务和随记，不要同时进行大批量文件拖放。

DeskBox 会先检查

- 备份结构是否受支持。
- 必需文件是否存在。
- 清单中的文件摘要是否匹配。
- 数据能否通过基本解析和安全检查。

通过后，恢复数据进入暂存区域，并使用能够跨重启继续的流程应用。恢复前还会创建保护性快照。

恢复会修改当前 DeskBox 配置和记录。重要状态下建议先手动导出当前数据，再恢复旧备份。

## 自动快照

DeskBox 最多保留 7 份每日自动快照。同一天已存在新快照时，不会因为每次打开设置就重复创建。

设置页面可以

- 查看快照时间、类型和状态
- 立即创建快照
- 删除受管理的快照
- 打开快照目录
- 识别不可读取或损坏的快照

自动快照是快速回退保护，不应成为唯一备份。硬盘损坏或整个用户目录丢失时，同盘快照也可能一起消失。

## 恢复前快照

每次应用恢复前，DeskBox 会创建恢复前快照，最多保留 5 份。

它的作用是，当恢复文件本身正确，但你后来发现选错版本时，仍然有机会回到恢复前状态。

## 附件的两种保存方式

### 关联原路径

DeskBox 只保存路径。优点是不重复占空间，缺点是原文件移动、改名、删除或外部磁盘断开后，附件会失效。

### 复制到 DeskBox

DeskBox 保存托管副本。优点是记录更稳定，缺点是占用本地数据目录空间，也需要跟随 DeskBox 备份管理。

建议

- 大型、稳定项目文件使用关联原路径。
- 微信、浏览器、临时目录文件使用复制。
- 重要内容同时保留正式项目备份。

## 附件健康检查

数据健康页可以扫描

- 缺失的关联原路径文件
- 记录存在但托管文件丢失
- 托管目录中存在但没有记录引用的孤立附件

扫描只报告，不会自动删除。

处理建议

- 原路径缺失，找到文件后重新关联，或确认记录可以删除。
- 托管文件缺失，从其他备份恢复，或移除无效引用。
- 孤立附件，先确认没有旧备份或未完成恢复依赖，再手动处理。

不要看到「孤立」就立即批量删除。异常退出、旧版本迁移和未完成恢复都可能留下需要判断的文件。

## 收纳根目录和应用数据目录

收纳根目录保存文件格子管理的普通文件夹，可以由用户选择。

应用数据目录保存配置、待办、随记、附件关系、托管附件和快照等内部数据，通常位于用户本地应用数据区域。

两者用途不同

- 收纳目录更像用户文件空间。
- 应用数据目录更像 DeskBox 的状态与数据库空间。

不要在 DeskBox 运行时手动编辑内部 JSON 或移动内部附件目录。需要迁移时，优先使用备份恢复和设置中的路径工具。

## 孤立收纳文件夹

如果格子配置丢失，但收纳根目录中的真实文件夹仍然存在，它会成为孤立收纳文件夹。

维护页面可以扫描并恢复为格子。恢复不会重新下载或转换文件，只是重建对应的 DeskBox 配置。

## 全局重置和功能格子重置

### 全局重置

恢复外观、交互等偏好设置，不应删除普通文件、待办或随记。语言、开机启动和功能格子开关等部分项目会按产品保护规则保留。

### 功能格子重置

针对待办、随记、音乐或天气等单独功能，可能清除对应数据和设置，需要二次确认。

两者不是同一个操作。

看到「重置」前，先读确认对话框中写的是偏好还是数据。

## JSON 损坏恢复

DeskBox 对关键 JSON 数据提供弹性读取和回退机制。发现主文件无法解析时，会尝试使用安全副本或恢复路径，而不是直接用空数据覆盖。

如果界面提示数据损坏

1. 不要反复新建同名内容。
2. 完全退出 DeskBox。
3. 先复制整个应用数据目录作为现场备份。
4. 使用维护工具或已知良好 ZIP 恢复。
5. 无法判断时保留文件并提交日志，不要自行删除。

## 升级前检查清单

重要版本升级前建议

1. 导出手动 ZIP 备份。
2. 确认备份文件大小不是 0，页面显示完整性正常。
3. 打开收纳根目录，确认普通文件可见。
4. 对映射的项目目录执行独立备份。
5. 完全退出 DeskBox，或让新版安装器自动关闭旧进程。
6. 覆盖安装后检查格子位置、待办、随记和附件。

## 推荐备份策略

- 日常依赖自动快照做短期回退。
- 每次大版本升级前导出手动 ZIP。
- 每周或每月把手动 ZIP 复制到另一块磁盘或云盘。
- 映射目录和重要收纳文件使用独立文件备份。
- 定期运行附件健康检查。

DeskBox 备份保护 DeskBox 的工作状态。完整数据安全仍然需要多位置备份。
