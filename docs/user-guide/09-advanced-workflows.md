# DeskBox 进阶工作流与特殊玩法

DeskBox 最有价值的用法，不是把每个功能都开启，而是让文件、待办和临时信息在同一个桌面动作中连接起来。

下面的方案都可以直接照着搭，也可以只取其中一部分。建议每次只尝试一种，使用一周后再调整。

> [图片占位：四种不同 DeskBox 桌面布局缩略图。分别标注收件箱、项目驾驶舱、内容创作和极简胶囊]

## 玩法一，桌面收件箱

适合每天从微信、浏览器、邮件和聊天软件接收大量文件的人。

### 搭建

创建三个收纳格子

1. 待处理
2. 等待回复
3. 本周归档

再放一个待办格子，只显示今天、重要和已完成。

### 使用

所有没有明确归属的新文件先进入「待处理」。不要在文件刚到时花时间决定最终目录。

需要别人补信息的文件移到「等待回复」，同时建立一条待办并关联该文件。处理完成的成果进入「本周归档」，每周固定一次移动到正式项目目录。

### 价值

它把「文件放在哪里」和「下一步做什么」分开。文件先有临时归宿，任务再负责推进，不需要把桌面当记忆工具。

> [动图占位：微信文件拖到待处理，建立关联待办，完成后文件移动到本周归档]

## 玩法二，项目驾驶舱

适合同时推进一个主要项目，希望桌面直接呈现项目上下文的人。

### 搭建

- 映射「项目目录」。
- 映射「项目素材」。
- 创建「输出交付」收纳格子。
- 待办只显示进行中、今天和本周。
- 随记固定项目目标、会议入口和常用回复。

把音乐和天气放进右侧组合胶囊栏，减少对主要工作区的占用。

### 使用

项目目录保持原结构，不为了 DeskBox 移动。当天工作文件从映射格子直接打开，输出结果拖到交付格子。需要跟进的文件关联到待办，临时决策写进随记。

### 价值

不用打开一个复杂仪表盘，桌面本身就是当前项目的入口。结束项目时，删除映射格子不会删除项目目录。

## 玩法三，内容创作流水线

适合设计师、视频创作者、公众号作者和内容运营。

### 搭建

创建或映射四个文件格子

- 选题与资料
- 原始素材
- 制作中
- 已导出

在原始素材和制作中开启自动叠放。

自定义规则示例

- 图片，`.png .jpg .jpeg .webp .gif`
- 视频，`.mp4 .mov .mkv`
- 设计源文件，`.psd .ai .fig`
- 字幕，`.srt .ass .vtt`
- 工程文件，按实际软件扩展名配置

随记固定品牌语气、标题模板和常用链接。待办用颜色区分脚本、设计、审核和发布。

### 使用

浏览器和聊天里找到的资料先进入选题与资料。参考图可以作为多图随记保存，正文写清来源和用途。制作中的工程文件使用 QuickLook 预览能够支持的格式，导出结果统一进入已导出。

### 价值

自动叠放负责降低文件视觉噪音，随记保存轻量上下文，待办负责流程推进。三者分别做擅长的事情。

## 玩法四，研究与阅读队列

适合学生、研究人员和需要长期收集资料的用户。

### 搭建

- 「待读」收纳格子，存放 PDF、网页导出和文档。
- 「当前课题」映射格子，连接真实研究目录。
- 随记显示记录和固定，最近剪贴板按需要开启。
- 待办显示今天、本周和重要。

### 使用

文章和 PDF 先进入待读，不要求立刻分类。每份真正开始阅读的资料建立一条待办，必要时把原文件作为附件关联。阅读过程中的临时摘录进入随记，确定有长期价值后再迁移到正式笔记系统。

### 价值

DeskBox 负责「还没进入知识库之前」的缓冲阶段，不和 Obsidian、OneNote、Zotero 等长期工具争夺职责。

## 玩法五，交付与回复清单

适合频繁给客户、同事或群聊发送文件的人。

创建一个「待发送」收纳格子和一个待办格子。每条任务写清接收人、渠道和截止时间，并关联要发送的多个文件。

复制任务时，DeskBox 会同时复制文字与附件路径。可以先把内容粘贴到聊天窗口检查，再从路径或格子拖出文件。

任务发送完成后标记完成，文件移入正式交付目录。

这个流程可以减少「文件已经做好，但忘了发」和「发了正文，漏了附件」两类常见问题。

## 玩法六，演示与共享屏幕隐私模式

开会共享屏幕前，不一定要关闭所有格子。

可以

- 开启胶囊隐私，隐藏待办与随记正文。
- 把内容模式改成仅图标和标题。
- 使用 F7 暂时隐藏全部格子。
- 将包含客户名称或私人内容的格子设为保持展开之外的安全状态。

隐私模式只改变收起状态展示，不是数据加密。共享屏幕时仍然应该检查展开窗口、通知和最近文件。

## 玩法七，右侧信息栏

把天气、音乐、待办和随记组成组合胶囊栏，贴在屏幕右侧。

推荐配置

- 方向自动
- 关键信息
- 保持同宽
- 悬停展开，平衡响应
- 标准动画
- 待办与随记开启隐私或摘要

标题中间用于展开，左右操作区域用于拖动和按钮。组合栏可以整体移动，格子顺序在设置中调整。

这种布局适合宽屏。屏幕较窄时，改为顶部横向栏或使用点击展开。

## 玩法八，完全不使用胶囊

胶囊不是必选项。

如果你的格子数量少、窗口尺寸已经合适，保持展开配合 F7 可能更高效。按 F7 统一显示和隐藏，省去逐个展开的动作。

高级功能的价值不是必须开启，而是给不同工作习惯更多选择。

## 玩法九，双屏分工

主屏放正在编辑的应用，副屏放 DeskBox 文件格子、待办和天气。

建议

- 格子按屏幕边角稳定放置。
- 经常查看的待办靠近两屏交界，但不要跨屏展开。
- 胶囊独立宽度只在屏幕空间确实需要时开启。
- 拔掉外接显示器前，先结束拖放、动画和详情编辑。

DeskBox 会根据显示器拓扑和位置锚点尝试恢复可见位置。分辨率和缩放经常变化时，定期确认格子没有依赖屏幕外空间。

## 玩法十，一周复盘而不是一次性装修

DeskBox 很容易让人产生「先把桌面布置完美」的冲动。

更有效的方法是每周复盘一次

1. 哪个格子从来没打开，删除或映射回资源管理器。
2. 哪个格子文件太多，开启叠放或拆分用途。
3. 哪个标签页从来不用，在设置中隐藏。
4. 哪条固定随记已经过期，取消固定。
5. 哪些关联附件原路径失效，运行健康检查。

桌面是工作流的结果，不是装修项目。
