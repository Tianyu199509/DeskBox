## 🔄 Phase 2 优先于 Phase 3 的原因

### 关键发现

通过对当前代码和设计文档的深度分析，我发现：

1. **方案 B 可能变得不再必要** ⭐
   - 方案 C 从根本上将 CPU 驱动改为 GPU 驱动
   - 批处理的动机（减少频繁的 SetWindowPos 调用）在方案 C 实施后不再存在
   - **可能性：** 方案 C 完成后，格子闪烁问题会显著改善甚至完全消失

2. **技术冲突风险** ⚠️
   ```
   原计划（A → B → C）❌
   
   ── 问题分析 ──
   • 方案 B 专注于"批处理多次 CPU 调用"
   • 方案 C 彻底移除"CPU 每帧调用"机制
   • 结果：方案 B 的大量工作可能是白费功夫
   
   新计划（A → C → B）✅
   
   ── 优势 ──
   • 先实施根本性解决方案
   • 观察剩余问题是否仍存在
   • 如需要再实施方案 B，避免重复劳动
   ```

3. **项目内已有最佳实践** ✅
   - `DetailPageTransitionHelper.cs` 已成功使用 GPU Animation
   - `QuickCaptureWidgetWindow` 已应用类似 Translation 技术
   - 证明方案 C 技术成熟度高

```csharp
// WindowUpdateBatcher - 核心类设计草案

/// <summary>
/// 批量处理窗口更新请求，实现原子性提交
/// </summary>
public class WindowUpdateBatcher
{
    private readonly Dictionary<long, PendingUpdate> _pendingUpdates = new();
    private readonly object _lock = new();
    
    /// <summary>
    /// 收集更新请求
    /// </summary>
    public void QueueUpdate(long widgetId, Rect newPosition, int durationMs)
    {
        lock (_lock)
        {
            _pendingUpdates[widgetId] = new PendingUpdate
            {
                WidgetId = widgetId,
                TargetPosition = newPosition,
                Duration = durationMs,
                QueuedAt = DateTime.Now
            };
            
            // 延迟批量提交（比如 16ms，约等于 60fps 间隔）
            StartBatchTimerIfNotRunning();
        }
    }
    
    /// <summary>
    /// 批量提交（原子性）
    /// </summary>
    private void FlushBatch()
    {
        List<PendingUpdate> batchCopy;
        
        lock (_lock)
        {
            if (_pendingUpdates.Count == 0) return;
            
            batchCopy = _pendingUpdates.Values.ToList();
            _pendingUpdates.Clear();
        }
        
        // 一次性提交所有更新（原子性操作）
        ApplyAllUpdates(batchCopy);
    }
    
    private void ApplyAllUpdates(List<PendingUpdate> updates)
    {
        // 策略 1: 使用双缓冲
        // 先在内存中计算所有最终状态
        
        // 策略 2: 批量调用窗口管理器
        foreach (var update in updates)
        {
            // 只调用一次而非每帧调用
            MoveWindowAtomic(update);
        }
    }
}

/// <summary>
/// 待处理的更新请求
/// </summary>
public class PendingUpdate
{
    public long WidgetId { get; set; }
    public Rect TargetPosition { get; set; }
    public int Duration { get; set; }
    public DateTime QueuedAt { get; set; }
}
```
