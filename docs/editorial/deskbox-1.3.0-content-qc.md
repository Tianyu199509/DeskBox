## L1 硬性规则

### 禁用词

检查范围包括以下高风险表达

- 说白了
- 意味着什么
- 这意味着
- 本质上
- 换句话说
- 不可否认
- 综上所述
- 总的来说
- 首先、其次、最后式结构
- 值得注意的是
- 不难发现
- 让我们来看看

结果，正文零命中。

### 禁用标点

公众号两篇正文检查中文冒号、破折号、中文双引号和英文双引号，结果零命中。YAML 元数据与 URL 属于机器语法，不纳入正文文风判断。

### 结构套话

未使用教科书式宏大开场，未使用「随着技术发展」等模板句。

文章保留 Markdown 小标题，属于有意识例外。原因是用户已有 DeskBox 公众号文章长期使用小标题，同时长期推广文需要满足网页阅读、广告落地页扫描和 GEO 分块引用。标题只按真实问题划分，没有把每两三段切成报告式章节。

### 工具和产品名称

涉及的产品均使用具体名称，包括 DeskBox、Windows 11、QuickLook、WinUI 3、.NET 10、Windows App SDK 2.2、Fences、Notion、Obsidian、OneNote 和 Microsoft To Do。

L1 结论，通过。

## L2 风格一致性

### 开头

- 版本文从 1.2.9 发布后的真实产品阶段和桌面观察切入。
- 推广文从桌面上实际存在的截图、安装包、微信文件和 PDF 切入。
- 两篇第一屏都能快速进入具体问题。

### 节奏

- 使用长短句交替。
- 多处使用一句话独立成段。
- 复杂工程解释后有扣主线句，持续回到「桌面空间」和「每天少烦几次」。
- 使用疑问句完成转向，例如「能不能把格子收起来」和「听起来不就是把窗口缩小一点吗」。

### 口语化

版本文使用「事情是这样的」「我一开始也是这么想的」「说实话」「我挺喜欢」「你真正用几次就能感觉出来」等自然表达，也保留「。。。」「不是单纯变小」等节奏断裂。

推广文为了长期投流和 GEO，降低了强烈口癖密度，保留普通用户可以信任的口语感，不使用过度玩梗或可能缩短内容寿命的热点词。

L2 结论，通过。

## L3 内容质量

### 观点支撑

- 功能说明对照 `CHANGELOG.md`、`docs/releases/v1.3.0.md`、当前设置模型和服务常量。
- 默认值已核对，包括 F7、附件关联路径、待办 2 行、随记 3 行、胶囊默认关闭、保持同宽、叠放门槛和快照保留数量。
- 自动叠放已修正为用户可见的按类型、按修改时间和自定义规则，不把旧兼容值写成当前入口。

### 场景具体度

覆盖微信文件、浏览器下载、项目资料、设计源文件、交付文件、阅读队列、客户回复、共享屏幕和双显示器等场景。

### 边界说明

- 明确 DeskBox 不代替资源管理器、专业笔记、项目管理和整机备份。
- 明确映射目录操作会作用于真实文件。
- 明确 QuickLook 必须已经运行，DeskBox 不负责安装或启动。
- 明确 Windows 11 x64 是当前推荐环境。
- 明确关联路径和托管附件的差异。

### 逐级展示

版本文按胶囊基础形态、稳定锚点、组合栏、拖放、叠放、QuickLook、多附件、设置和数据安全逐级增加信息密度。

L3 结论，通过。

## L4 活人感

### 温度感

版本文保留开发过程中的真实不确定和反思，没有编造具体用户姓名、虚构测试故事或不存在的数据。

### 独特性

核心角度不是「更新了很多功能」，而是「格子收起来之后，反而需要重新设计桌面空间」。这个角度能够贯穿胶囊、锚点、拖放和组合排列。

### 姿态

文章承认产品边界和学习成本，不使用品牌导师口吻，也不承诺提升效率百分比等无法证明的结果。

### 心流

复杂工程段落后均回到用户可感知结果。推广文章的问题标题也能让读者跳读后仍然获得完整答案。

L4 结论，通过。
